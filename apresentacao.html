Contexto e Problema

Entrega 1 · 0,5pts
📍

O Contexto Real

Uma loja de informática que precisa controlar seu dia a dia de forma digital. O atendente lida com clientes, produtos, estoque, vendas e notas fiscais — tudo manualmente ou em planilhas.

🛒 Vendas 📦 Estoque 🤝 Clientes 🧾 Nota Fiscal
⚠️

Os Problemas

  • Sem controle de estoque automatizado
  • Notas fiscais emitidas à mão
  • Sem relatório de vendas por período
  • Dois perfis de acesso (Dono / Funcionário) sem separação
  • Impossível consultar histórico de clientes
🎯

Objetivo Principal

Digitalizar a operação completa da loja com um sistema web moderno, acessível e seguro.

👥

Público Alvo

Proprietário da loja (DONO) e seus funcionários (FUNCIONÁRIO) com acessos distintos e controlados.

🌐

Solução Proposta

Sistema Web com Frontend moderno consumindo uma API REST Django, protegida com JWT e hospedada em nuvem.

📋

Diagrama Lógico do Banco

Entrega 1 · 0,5pts
-- Tabela: core_usuario CREATE TABLE core_usuario ( id SERIAL PRIMARY KEY, username VARCHAR(150) UNIQUE, email VARCHAR(254), password VARCHAR(128), perfil VARCHAR(50) CHECK (perfil IN ('DONO','FUNCIONARIO')) ); -- Tabela: core_cliente CREATE TABLE core_cliente ( id SERIAL PRIMARY KEY, nome VARCHAR(255) NOT NULL, cpf VARCHAR(14) UNIQUE NOT NULL, email VARCHAR(254), telefone VARCHAR(20), endereco VARCHAR(255) );
-- Tabela: core_produto CREATE TABLE core_produto ( id SERIAL PRIMARY KEY, nome VARCHAR(255) NOT NULL, descricao TEXT, preco DECIMAL(10,2) NOT NULL, quantidade_estoque INTEGER DEFAULT 0, estoque_minimo INTEGER DEFAULT 5, imagem TEXT ); -- Tabela: core_venda CREATE TABLE core_venda ( id SERIAL PRIMARY KEY, data TIMESTAMPTZ, valor_total DECIMAL(10,2), forma_pagamento VARCHAR(20), cliente_id INTEGER REFERENCES core_cliente(id), usuario_id INTEGER REFERENCES core_usuario(id) );
-- Tabela: core_itemvenda CREATE TABLE core_itemvenda ( id SERIAL PRIMARY KEY, quantidade INTEGER NOT NULL, valor_unitario DECIMAL(10,2), venda_id INTEGER REFERENCES core_venda(id) ON DELETE CASCADE, produto_id INTEGER REFERENCES core_produto(id) ON DELETE RESTRICT );
🔑 Chaves Primárias — SERIAL (Auto-Increment)
🔗 Foreign Keys — com RESTRICT para integridade
✓ 3ª Forma Normal — sem redundâncias

✅ Exceções Implementadas

  • ❌ Estoque insuficiente ao vender
  • ❌ Excluir cliente com vendas vinculadas
  • ❌ Excluir produto com vendas vinculadas
  • ❌ CPF duplicado ao cadastrar cliente
  • ❌ Acesso sem token (401 Unauthorized)
  • ❌ Acesso de Funcionário a rotas de Admin (403)

🔄 Handler Global no Frontend

Todos os erros são capturados em api.js com try/catch e exibidos ao usuário via SweetAlert2 — modais bonitos com mensagem amigável, sem crash na tela.
